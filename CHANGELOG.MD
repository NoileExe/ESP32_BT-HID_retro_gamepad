# Changelog

Все значимые изменения в проекте документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
версионирование соответствует [Semantic Versioning](https://semver.org/lang/ru/).

> **Примечание для разработчиков:**
> - Аппаратные версии плат (HW) и версии прошивки (SW) версионируются независимо
> - Переключение между аппаратными конфигурациями осуществляется через дефайны:
>   - `ONLY_DIGITAL_GAMEPAD` — поддержка только цифровых кнопок (конфигурация BLE HID-отчёта)
>   - `ANALOGUE_GAMEPAD` — поддержка аналоговых стиков и триггеров (конфигурация BLE HID-отчёта)
>   - `USE_MAX1704X_FUEL_GAGE` — использование модуля MAX1704x вместо делителя напряжения на резисторах
> - Проект находится в стадии активной разработки (pre-release 0.x.y)
> - Стек разработки: C++17, VS Code + PlatformIO, Arduino Framework (обусловлено использованием библиотеки [ESP32-BLE-CompositeHID](https://github.com/lemmingDev/ESP32-BLE-CompositeHID)).

## [0.4.0] - 2026-06-15

### Added
- Альтернативный резисторному делителю способ измерения напряжения и процента заряда через модули на базе микросхем серии **MAX1704x** (SDA_PIN = 0, SCL_PIN = 1) — **HW v1.2**.
- Переключение между измерением на делителе и MAX1704x посредством дефайна `USE_MAX1704X_FUEL_GAGE` (`true`/`false`) в **gamepadConfig.hpp**.
- Индикация ошибки I2C при запуске/пробуждении: при отсутствии связи с модулем MAX1704x выполнение программы замораживается с постоянным миганием светодиода.

### Changed
- Питание модуля MAX1704x осуществляется напрямую от LDO платы ESP32-C3 SuperMini (требуется перерезание дорожки между `BAT+` и `VCC` на плате модуля MAX1704x).
- Определение низкого заряда теперь базируется на проценте, а не на абсолютном значении напряжения.
- Упрощён основной цикл: логика опроса кнопок, отлова комбинаций и отправки HID-отчётов вынесена в отдельный метод.

## [0.3.0] - 2026-06-15

### Added
- Программное определение процента заряда аккумулятора на основе таблицы напряжений с шагом 2% (значения сформированы ИИ).
- Передача (реклама) процента заряда аккумулятора на BLE-хост.

## [0.2.0] - 2026-06-15

### Added
- Измерение напряжения аккумулятора на резисторном делителе, подключённом к GPIO4 (ADC1) — **HW v1.1**.
- Светодиодная индикация низкого и критически низкого заряда.
- Блокирующий цикл при достижении критически низкого заряда (до выхода из критической зоны, например, начала зарядки).
- Закомментированный отладочный вывод напряжения в Serial для калибровки показаний ADC.

### Changed
- Выполнено единообразное именование методов и переменных в `main.cpp` (например, `btstatus_led` → `btstatusLed`).

## [0.1.0] - 2026-06-15

### Added
- **Первая релизная (жизнеспособная) версия**: NES-геймпад на базе платы ESP32-C3 SuperMini.
- Контроль наличия BLE-сопряжения со светодиодной индикацией:
  - режим поиска — мигание;
  - наличие сопряжения — постоянное свечение.
- Базовые настройки BLE-устройства: приём, согласование параметров и отправка HID-отчётов о нажатиях кнопок.
- Поддержка различных режимов для кнопок действий (A, B, C, X, Y, Z L1, R1, L3, R3): **Standard / Turbo / Slow**.
- Смена режимов кнопок удержанием комбинаций кнопок со световой индикацией текущего режима.
- Универсальный интерфейс для Generic и XBox (One S / Series X) геймпадов.
